# Проект базы данных на Python

## Новые возможности

### 1. Обработка ошибок
Все функции работы с базой данных теперь защищены декоратором `@handle_db_errors`, который:
- Перехватывает KeyError (несуществующие таблицы)
- Перехватывает ValueError (ошибки валидации)
- Перехватывает FileNotFoundError (отсутствие файлов)
- Предотвращает аварийное завершение программы

### 2. Подтверждение опасных операций
Опасные операции теперь требуют подтверждения:
- `@confirm_action("удаление таблицы")` - для drop_table
- `@confirm_action("удаление записи")` - для delete
Пользователь должен ввести 'y' для подтверждения операции.

### 3. Замер производительности
Медленные операции замеряют время выполнения:
- `@log_time` - для select и insert
- Результат выводится в консоль в формате "Функция <имя> выполнилась за X.XXX секунд"

### 4. Кэширование запросов
Реализовано интеллектуальное кэширование:
- Результаты одинаковых запросов `select` кэшируются
- При повторном запросе с теми же параметрами используется кэш
- Кэш очищается при перезапуске программы

@log_time
